# Readme Maker Skill

## When to Use
- New projects need documentation
- Existing projects lack clear README
- Want to improve project discoverability
- Need to communicate project purpose to others

## Procedure
1. Analyze project structure and purpose
2. Identify key information to include
3. Write clear, concise sections
4. Format with proper markdown structure

## Output Format

**Project Title**
[Clear, descriptive project name]

**Description**
[Brief overview of what the project does]

**Installation**
[Step-by-step setup instructions]

**Usage**
[How to use the project]

**Contributing**
[How others can contribute]

## README Formula

**Title + Description + Installation + Usage + Contributing + License**

## Examples

**Bad:**
> "My Project"

**Good:**
> "# My Awesome Project\n\nA tool that helps developers create better documentation automatically.\n\n## Installation\n`npm install my-awesome-project`\n\n## Usage\n`my-awesome-project --input file.md`\n\n## Contributing\nPull requests welcome!"

---

**Bad:**
> "Code"

**Good:**
> "# Data Processor\n\nProcesses and transforms JSON data files.\n\n## Features\n- Validate JSON structure\n- Transform data formats\n- Export to multiple formats\n\n## Quick Start\n1. Install: `pip install data-processor`\n2. Run: `data-processor input.json output.csv`"
